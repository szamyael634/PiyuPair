# PiyuPair System Features

## Core Platform

- Multi-role tutoring marketplace with Student, Tutor, and Admin roles.
- Supabase-backed authentication and session handling.
- Edge Function API for all business operations.
- KV-style data storage for application entities.

## Authentication and Onboarding

- Email/password sign up and login.
- Facebook OAuth sign in and sign up.
- OAuth profile sync to create first-time app profiles automatically.
- Role-based profile creation (student or tutor).
- Tutor approval workflow (pending until admin approval).
- Admin bootstrap command for first admin creation (`npm run bootstrap:admin`).

## Student Features

- Student dashboard with session, application, and activity stats.
- Browse tutors with filtering by search text, subject, rating, and rate.
- Tutor profile view with qualifications, subjects, rates, and ratings.
- Apply for tutoring sessions (online or in-person).
- Find class by class/session ID and open classroom directly.
- Upload certificate and grade documents.
- Suggested tutors after low grade uploads.
- Real-time session participation via classroom and video call routes.
- Profile editing (name, phone, bio, current grade, enrolled subjects).

## Tutor Features

- Tutor dashboard with pending applications and active sessions.
- Accept or decline student applications.
- Session/classroom access and video call entry.
- Earnings and performance stats.
- Profile editing (subjects, qualifications, hourly rate in PHP, experience, discount).
- Approval pending state with controlled access until admin approval.

## Admin Features

- Admin dashboard with platform-wide stats.
- View and process tutor approval requests.
- View tutor credentials and uploaded proof documents.
- Revenue and commission monitoring.
- Commission analytics by match-quality band (high, medium, low).
- Average commission rate tracking.
- Top-rated and top-performing tutor insights.

## Matching and Commission Business Model

- Tutor matching score generated by algorithm (0-100).
- Matching considers subject fit, rating quality, and session experience.
- Dynamic platform commission based on match score:
  - High confidence match -> lower commission
  - Lower confidence match -> higher commission
- Commission range implemented at 5% to 10%.
- Payment records include:
  - match score
  - commission rate
  - commission amount
  - tutor net amount

## Payments and Ratings

- Payment processing route with automated commission split.
- Tutor net earnings updates after payments.
- Student tutor-rating and review submission.
- Auto-updated tutor average ratings and review counts.

## Messaging

- One-to-one messaging between users.
- Conversation listing and message history.
- Unread state tracking per conversation.

## Classroom and Content

- Session management endpoints.
- Session activity posting and retrieval.
- Assignment/material/submission activity types.
- Student submission upload flow for activities.
- AI-style submission risk scoring on activity submissions.

## Learning Experience Enhancements

- Customizable learning path creation per student.
- Module-level progress tracking and completion state.
- Learning path progress analytics (completion %, average score, module completion count).
- Interactive assessment engine (create, fetch, submit, auto-score).
- Gamified progression with XP, levels, and achievement badges.

## Scheduling and Availability

- Tutor availability slot publishing API.
- Student booking request flow using tutor availability slots.
- Booking list endpoints filtered by role (student, tutor, admin).
- Designed for real-time scheduling integrations on frontend.

## Credential and Output Intelligence

- Credential consistency checks for uploaded certificates and grades.
- Field-to-document text matching signals (subject/type/grade/name).
- Verification status returned with mismatch diagnostics.
- AI detection metadata on submitted outputs (risk score and flags).

## Multi-Language

- Per-user preferred language setting endpoint.
- Localized message dictionary endpoint (`en`, `fil`, `es` fallback to `en`).

## API Security and Access Rules

- Authorization checks on protected routes.
- Role-based access in admin endpoints.
- Session-by-ID access control:
  - students can access only their own sessions
  - tutors can access only their own sessions
  - admins can access all sessions

## Currency and Localization

- Philippine Peso formatting across key financial UI screens.
- PHP currency labels in rate input fields.
